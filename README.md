## 📦 Como Entregar (OBRIGATÓRIO)

A entrega será feita **exclusivamente via Pull Request (PR)** neste repositório.

### 🔀 Passo a passo

1. Faça um **fork** deste repositório  
2. Crie uma branch com seu nome:
   feat/seu-nome
3. Crie uma pasta com seu nome dentro do projeto
4. Adicione seus arquivos dentro dessa pasta
5. Faça commits organizados
6. Abra um Pull Request para este repositório

### 🧾 O que escrever no Pull Request

No seu PR, inclua:
- O que você desenvolveu
- Dificuldades encontradas
- O que aprendeu durante o processo

### ⚠️ Regras Importantes
- ❌ NÃO envie arquivos desnecessários
- ❌ NÃO altere arquivos de outros candidatos
- ❌ NÃO envie arquivos fora da sua pasta
- ✅ Use código limpo e bem organizado
- ✅ Indente corretamente
- ✅ Nomeie arquivos e classes de forma clara

## 🌐 Desafio – Front-End

Crie uma página de apresentação do Calama Devs.

### 🎯 Objetivo
Simular uma landing page simples do time.

### 📋 Requisitos mínimos
- HTML semântico
- CSS organizado
- Layout minimamente responsivo
- JavaScript com pelo menos uma interação

### 💡 Funcionalidade obrigatória
Adicionar um botão com alguma ação, por exemplo:
- Exibir mensagem motivacional
- Alterar conteúdo da página
- Simular inscrição

**Exemplo:**
Botão: "Quero fazer parte" → exibe mensagem na tela

### 📁 Estrutura obrigatória

Você deve criar uma pasta com seu nome:
```
/seu-nome/
  ├── index.html
  ├── css/
  │   └── style.css
  ├── js/
  │   └── script.js
```
### 🛠️ Boas práticas esperadas
- Separação correta entre HTML, CSS e JS
- Código legível e organizado
- Evitar repetição de código
- Uso de nomes claros para classes e arquivos

### ⭐ Diferenciais (não obrigatórios)
- Uso de Flexbox ou Grid
- Animações simples
- Design mais elaborado
- Código reutilizável

### 🔀 Commits (OBRIGATÓRIO)
Faça pelo menos 3 commits, por exemplo:
- **feat**: adiciona estrutura HTML inicial
- **style**: adiciona estilos da página
- **feat**: implementa botão interativo

Evite commits genéricos como:
- update
- alteração

### 📊 Critérios de Avaliação
- Organização do código ->	25%
- Qualidade do HTML/CSS/JS -> 25%
- Uso de Git (commits + PR) -> 20%
- Atendimento aos requisitos -> 20%
- Capricho e extras -> 10%

### ❌ Motivos de desclassificação
- PR fora do padrão
- Arquivos desnecessários
- Código desorganizado
- Não seguir a estrutura
- Apenas 1 commit

### 💡 Observação importante

- Vídeos / tutoriais
- IA
**Mas precisa entender o que está fazendo!**
Você poderá ser questionado(a) sobre seu código.
